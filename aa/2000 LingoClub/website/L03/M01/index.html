Property Variables.

Reusable Behavior Script:

The built-in proprty spriteNum is a property variable is automatically set to the number of the sprite that the script is attached to. This lets you use the same script with different sprites.

Attaching script can be attached to any sprite and will cause it to move left on the stage.

-- Move left
property spriteNum

on exitFrame
    sprite(spriteNum).locH = sprite(spriteNum).locH + 1
    if sprite(spriteNum).locH > 100 then
        sprite(spriteNum).locH = 10
    end if
end

Making Your Own Property Variables.

speed example. the property speed is a made up name. The value in speed stays around until the sprite ends in the score. Here the sprite will move at a random speed between 5 and 10 pixels for every exitFrame

-- Move left behavior script with random speed.

property spriteNum
property speed

on exitFrame
    sprite(spriteNum).locH = sprite(spriteNum).locH + speed
    if sprite(spriteNum).locH > 100 then
        sprite(spriteNum).locH = 10
    end if
end

on beginSprite
    speed = random(5,10)
end

Know The Lingo

!!@ Notes

property spriteNum is setup by Lingo for every Script attached to a Sprite.

spriteNum is a pre-defined property variable. You can define your own property variables.

Local, global, parameter and property variables.
Differ by how long they stay around.
Property variables stay around for the line time of the object. In the case of script attached to a sprite, the property variables stays around until the sprite ends.

Show box to visualize the location of variable.
